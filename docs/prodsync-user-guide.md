## 1. 技能定位

Sloth-ProdSync-Eido 是面向 **快消品（肉制品）行业** 的深筹 · 产销协同智能副驾，服务于 L2 计划经理和 L3 车间计划员。

**核心理念**：

| 原则 | 含义 |
|------|------|
| 约束满足 > 最优化 | 在多约束下找"可行且好"的方案，不追求数学最优 |
| 参谋 > 夺权者 | 给出建议 + 影响分析 + 量化依据，由人类做最终决策 |
| 数字说话 | 每个判断都有量化支撑，杜绝"影响不大"等模糊表述 |
| 班次级建议 | 非分钟级实时控制，适合对话式咨询场景 |

**能力边界**：

- 调度粒度：**班次级**（日/班次），非分钟级实时控制
- 执行方式：输出推荐方案供人工确认，不自动下发指令
- 数据来源：用户上传 Excel 模板 + 对话追加变更
- 参数体系：370+ 可配参数，从 `config/enterprise-profile.yaml` 加载
- 约束引擎：十一项约束检查 + DAG 五层级联传导

## 2. 安装与初始化

### 2.1 安装步骤

1. 在 QoderWork 中安装 Sloth-ProdSync-Eido 技能
2. 确保 Python 3.8+ 已安装（用于运行计算脚本）
3. 安装 Python 依赖：
   ```bash
   pip install openpyxl pyyaml
   ```

### 2.2 企业配置初始化

首次使用前建议完成企业参数配置，有三种级别可选：

| 级别 | 适用场景 | 所需时间 | 参数覆盖 |
|------|---------|---------|---------|
| L1 快速配置 | 小企业 / 快速体验 | 5-10 分钟 | 核心参数（~30项） |
| L2 标准配置 | 中型企业 | 30-60 分钟 | 主要参数（~120项） |
| L3 深度配置 | 大型企业 / 精确需求 | 2-4 小时 | 全部参数（370+项） |

配置方式：对话中说 **"企业配置引导"** 或 **"初始化企业参数"**，系统将引导你逐项完成。也可以直接编辑 `config/enterprise-profile.yaml` 文件。

### 2.3 数据准备

准备以下 Excel 模板（模板文件位于 `templates/` 目录）：

| 模板 | 文件 | 关键字段 |
|------|------|---------|
| 订单表 | `orders-template.xlsx` | 订单号/产品/数量/交期/客户/优先级 |
| 产能表 | `capacity-template.xlsx` | 产线/日产能/当前利用率/计划停机/CIP时间 |
| BOM 配方表 | `bom-template.xlsx` | 成品/原料/配比/替代品/最小批量 |
| 库存快照 | `inventory-template.xlsx` | 物料/数量/入库时间/保质期/存储位置 |

如果只想快速体验，可使用 `templates/sample-data-meat.xlsx` 样例数据（含12订单/8产线/26BOM/16库存）。

## 4. 使用流程指南

### 4.1 标准工作流

```
步骤1：上传数据
  上传 Excel 模板（订单/产能/BOM/库存）
  → 系统自动解析、校验、输出确认摘要

步骤2：描述需求
  例："帮我评估一下这个插单"
  例："做一下本周的排产建议"
  例："对比旺季保供和淡季利润两个方案"

步骤3：查看结果
  → 系统输出约束检查结果 + 影响分析 + 量化建议
  → 附带行动清单和验证节点

步骤4：确认/调整
  "采纳方案A" / "调整产线分配" / "换个方案看看"

步骤5：追加变更（可选）
  "今天B厂2号线故障停产4小时"
  → 系统自动触发受影响约束的重新检查
```

### 4.2 插单评估流程

1. 上传或口述插单需求（产品、数量、交期、客户、优先级）
2. 系统执行 **十一项约束检查**（按 DAG 五层级联传导）：
   - L1 外部输入层：原料到货、人员出勤
   - L2 工艺层：物料供给、CIP清洗、批次约束、工艺锁
   - L3 资源层：产能利用、合规检验
   - L4 交付层：冷链温控、物流排车
   - L5 终端汇聚层：时间窗综合判定
3. 计算综合紧急指数（参见 [urgency-index.md](urgency-index.md)）
4. 输出标准影响评估卡（参见 [output-formats.md](output-formats.md)）
5. 给出采纳/调整/拒绝建议及量化依据

### 4.3 What-if 情景推演流程

1. 描述需要对比的场景或选择标准战略模式：

   | 模式 | 核心策略 | 典型场景 |
   |------|---------|---------|
   | 旺季保供 | 交付率优先 | 春节备货、中秋旺季 |
   | 淡季利润 | 成本优先 | 夏季淡季 |
   | 临期清库 | 库存周转优先 | 季节末尾 |

2. 系统构建场景并完成级联约束评估
3. 四维评分对比（交付 35% / 成本 25% / 库存 25% / 风险 15%）
4. 输出对比表 + 每维度差值/优胜/洞察 + 综合推荐
5. 可使用"一键三模式对比"快速生成全部方案比较

### 4.4 可视化看板流程

1. 完成排产分析后，说"生成可视化看板"或"生成 HTML 看板"
2. 系统生成四种可视化：
   - **产线甘特图**：各产线每日/班次作业序列
   - **物料平衡桑基图**：屠宰 -> 部位 -> 产线的流向与量
   - **库存预警热力图**：各物料库存水位（红/黄/绿/蓝四级）
   - **紧急指数仪表盘**：在产订单紧急度排名 + 雷达图
3. HTML 看板为自包含文件（约 60KB），浏览器直接打开

## 5. 最佳实践

### 5.1 数据准备

- **使用标准模板**：严格按照 `templates/` 目录下的模板格式填写数据，避免格式校验失败
- **样例数据先行**：首次使用时先上传 `sample-data-meat.xlsx` 熟悉流程
- **数据完整性**：尽量填写所有字段，缺失字段会被标为假设值（带 ⚠️ 标记）

### 5.2 约束引擎

- **不要跳过约束检查**：即使时间紧迫，十一项约束检查是铁律，不可省略
- **关注级联传导**：一个约束的变化会沿 DAG 传播到下游约束，注意级联影响
- **留意假设值**：带 ⚠️ 标记的数字表示使用了假设值，请及时确认或替换为真实数据

### 5.3 企业配置

- **循序渐进**：建议从 L1 快速配置开始，逐步升级到 L2/L3
- **定期校准**：每季度核实关键参数（出成率、CIP时间、产能上限）是否需要更新
- **备份配置**：修改 `enterprise-profile.yaml` 前做好备份

### 5.4 排产决策

- **多方案对比**：重大决策时至少对比两个方案，使用 what-if 引擎量化差异
- **闭环跟踪**：认真执行行动清单中的验证节点和复盘检查点
- **及时追加变更**：设备故障、到货延迟等变更要及时通过对话追加，保持数据模型准确

## 6. 常见问题 (FAQ)

### Q1：上传 Excel 后提示数据校验不通过，怎么办？

系统会给出具体的修复建议（哪一行、哪个字段、应该是什么格式）。请按提示修正后重新上传。常见问题包括：日期格式不统一、数量字段为文本而非数字、必填字段缺失。建议使用 `templates/` 目录下的标准模板。

### Q2：约束检查结果中出现很多 ⚠️ 假设值，这正常吗？

这说明部分数据缺失，系统使用了行业标准默认值或经验值进行推算。带 ⚠️ 的结果仅供参考，建议尽快补充真实数据（如实际出成率、实际 CIP 时间等）以提高分析准确度。

### Q3：可以同时评估多个插单吗？

可以。你可以一次上传或口述多个插单需求，系统会逐一评估并给出综合影响分析。但建议按优先级排序，因为先评估的插单会改变后续插单的约束环境。

### Q4：情景推演的四维评分权重可以自定义吗？

可以。默认权重为交付 35% / 成本 25% / 库存 25% / 风险 15%。你可以在对话中说"调整评分权重：交付40%、成本30%、库存20%、风险10%"来自定义。也可以选择三种标准战略模式（旺季保供/淡季利润/临期清库），每种模式有预设的权重方案。

### Q5：如何查看约束之间的级联传导路径？

系统在输出约束检查结果时会包含 `propagation_log`，记录了人可读的级联轨迹。例如"原料到货延迟2h -> 开工推迟2h -> 产能利用率降低12% -> 时间窗压缩，交付风险升高"。你也可以说"展示级联传导详情"获取完整的传导链路。
