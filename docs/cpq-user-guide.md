## 技能定位

Sloth-CPQ-Eido（深算 · 报价管理）是一个运行在 QoderWork 平台上的 **AI 智能报价助手**（Configure-Price-Quote），专为制造业销售工程师设计。它能在几分钟内完成过去需要数小时的报价工作，通过自然语言理解客户需求，自动提取关键参数，基于行业知识库和成本模型生成多方案报价对比。

**适用人群：**

- 装备制造、通用机械加工、电子制造行业的销售工程师
- 需要快速估价和交期评估的业务人员
- 管理报价版本和偏差学习的销售管理者

**覆盖行业：**

| 行业 | 知识文件 | 典型产品 |
|------|---------|---------|
| 装备制造 | `equipment-knowledge.md` | 液压机、压力机、冶金设备、非标产线 |
| 通用机械加工 | `machinery-knowledge.md` | 轴类、盘类、箱体、钣金、框架结构件 |
| 电子制造 | `electronics-knowledge.md` | PCB、PCBA、SMT 贴装、线缆组件、整机组装 |

**核心价值：**

- 六步标准化报价工作流，确保报价质量和一致性
- 多场景报价比选（优先交期/优先利润/均衡方案）
- 数据信心分级（A/B/C），每项关键数据标注来源
- 盈亏交通灯可视化（绿 >15%、黄 8%-15%、红 <8%）
- 偏差学习机制，持续优化报价准确性

## 安装与初始化

### 安装方式

将技能目录安装到 QoderWork：

```bash
cp -r v1.0-extension/sloth-cpq-eido/ ~/.qoderwork/skills/sloth-cpq-eido/
```

如需 MCP 集成管理（运维人员），还可安装辅助 Skill：

```bash
cp -r v0.5-mcp/cpq-integration-helper/ ~/.qoderwork/skills/cpq-integration-helper/
```

### 数据准备（可选）

为获得更精确的报价，建议准备以下 CSV 数据模板：

| 模板文件 | 用途 | 路径 |
|----------|------|------|
| `material_data.csv` | 物料成本与货期数据 | `templates/material_data.csv` |
| `project_history.csv` | 历史项目工时数据 | `templates/project_history.csv` |
| `workcenter_calendar.csv` | 瓶颈设备产能日历 | `templates/workcenter_calendar.csv` |
| `quote_version_log.csv` | 报价版本记录 | `templates/quote_version_log.csv` |

### 数据来源优先级

技能按以下优先级获取数据（渐进式降级）：

1. **MCP 实时数据**：通过 MCP 工具获取最新价格和产能（信心等级 A）
2. **CSV 模板数据**：读取 `templates/` 目录中的参考数据（信心等级 B）
3. **对话收集**：在对话中向用户询问关键数据（信心等级 B）
4. **行业默认值**：使用内置行业基准，标注精度降级警告（信心等级 C）

## 核心功能

### 六步报价工作流

| 步骤 | 名称 | 说明 |
|:---:|------|------|
| 1 | 需求解析与追问 | 从自然语言提取关键参数，追问缺失项（每次不超过 5 个问题） |
| 2 | 成本穿透估算 | 按成本计算指南执行多维度成本拆解 |
| 3 | 交期可行性验证 | 从物料、产能、设计三个维度交叉验证交期 |
| 4 | 合同条款风险扫描 | 检查延期罚金、质保期、付款条件、出口认证等风险 |
| 5 | 多场景报价生成 | 输出至少 2 个方案（优先交期/优先利润/均衡），含利润交通灯 |
| 6 | 报价版本管理 | 保存报价快照，与历史报价对比 |

### 触发命令示例

| 命令 | 说明 |
|------|------|
| `客户需要一台 630 吨四柱液压机，西门子控制，力士乐液压，交期 90 天，帮我估个价` | 装备制造询价 |
| `有一批 40Cr 传动轴，直径 80mm 长 500mm，IT7 公差，200 件，15 天交货，报个价` | 机械加工询价 |
| `4 层 FR-4 板，100x80mm，280 个贴片点，首批 500 块，要代采元器件，报价` | 电子制造询价 |
| `上次 XX 客户的液压机项目实际成本出来了，帮我做偏差分析` | 偏差学习回写 |

### 行业自动识别

技能会根据产品描述中的关键词自动加载对应行业知识库，无需手动选择行业。例如：

- 提到"液压机""压力机""冶金设备" --> 自动加载装备制造知识库
- 提到"传动轴""箱体""钣金" --> 自动加载通用机械加工知识库
- 提到"PCB""SMT""贴装" --> 自动加载电子制造知识库

### 特殊风险识别

当检测到以下特殊条件时，技能会立即发出警告：

- 防爆要求
- 出口认证
- 极寒条件
- 军工标准

### Python 自动化脚本

| 脚本 | 用途 | 使用方式 |
|------|------|----------|
| `parse_bom_csv.py` | 解析上传的 BOM CSV，交叉匹配物料数据 | `python scripts/parse_bom_csv.py <bom_file> <material_data_file>` |
| `calculate_cost.py` | 执行成本穿透计算 | `python scripts/calculate_cost.py --material <金额> --labor <金额> --delivery <天数>` |
| `validate_template.py` | 校验 CSV 文件格式合规性 | `python scripts/validate_template.py <file> --type material\|project\|workcenter` |
| `search_similar.py` | 搜索相似历史项目 | `python scripts/search_similar.py --type <产品类型> --tonnage <吨位> --data <csv路径>` |

## 工作流程指南

### 典型工作流：完整报价流程

```
步骤 1: 输入询价需求
  > 客户需要一台 630 吨四柱液压机，工作台 1200x1200mm，
    西门子控制系统，力士乐液压，交期 90 天，帮我估个价。

步骤 2: 回答追问（如有）
  技能会追问缺失的关键参数，如付款条件、运输安装要求等
  > 付款方式 3-3-3-1，需要安装调试，交付地点广东佛山

步骤 3: 查看成本穿透报告
  技能输出材料成本、人工成本、外协成本、管理费用等明细

步骤 4: 查看交期验证结果
  技能从物料、产能、设计三维度验证交期可行性

步骤 5: 查看多场景报价
  至少 2 个方案，每个标注利润交通灯和风险等级

步骤 6: 选择方案或调整参数
  > 方案一的利润太低了，如果把控制系统换成汇川呢？

步骤 7: 保存报价版本
  > 保存这个报价，标记为 V1
```

### 典型工作流：偏差学习

```
步骤 1: 提供实际成本数据
  > 上次报价的 630 吨液压机项目交付完了，实际材料成本 85 万，
    人工成本 32 万，外协 12 万

步骤 2: 查看偏差分析
  技能对比报价估算 vs 实际成本，计算偏差率

步骤 3: 确认风险系数更新
  > 确认更新装备制造类的材料风险系数

步骤 4: 后续报价自动参考
  在后续类似询价中，技能会主动提示历史偏差教训
```

### 报价输出说明

每个报价方案包含以下内容：

| 输出项 | 说明 |
|--------|------|
| 推荐对外报价（含税） | 建议的客户报价 |
| 预计净利润率 | 带交通灯标识：🟢 >15%、🟡 8%-15%、🔴 <8% |
| 交期风险等级 | High / Medium / Low |
| 核心执行要点 | 该方案需要执行的关键动作 |
| 最大风险项 | 该方案的最大潜在风险 |
| 因子明细表 | 材料因子、复杂度因子、精度因子、批量因子及选择依据 |
| 假设与排除项 | 本次报价的所有假设条件和未包含项目 |
| 数据信心评级 | A（全实时）/ B（部分缓存）/ C（使用默认值） |

## 最佳实践

1. **提供完整需求描述**：在第一次对话中尽可能提供完整的产品规格、数量、交期要求和付款条件，减少追问轮次，提高报价效率。

2. **准备 CSV 数据模板**：将公司实际的物料成本、历史项目数据填入 CSV 模板，可以显著提高报价精度（从信心等级 C 提升到 B）。

3. **关注利润交通灯**：红灯（<8%）方案需要特别谨慎，建议与管理层讨论后再报价。黄灯（8%-15%）方案可考虑优化成本结构。

4. **定期做偏差学习**：每个项目交付完成后，将实际成本数据回写，让技能持续学习和优化风险系数。长期积累可以大幅提升报价准确性。

5. **善用版本管理**：对同一客户的多次报价使用版本管理功能，方便对比不同方案的差异和演变。

6. **注意特殊风险提示**：当技能发出防爆、出口认证等特殊风险警告时，务必在报价中充分考虑额外成本和时间。

## 常见问题（FAQ）

### Q1: 技能如何自动识别产品属于哪个行业？

技能通过分析用户输入中的产品描述关键词来自动匹配行业知识库。例如"液压机""压力机"会匹配装备制造，"PCB""SMT"会匹配电子制造。如果关键词不够明确，技能会主动询问确认。

### Q2: 没有 MCP 工具和 CSV 数据也能使用吗？

可以。技能采用渐进式数据降级策略：没有 MCP 工具时使用 CSV 模板，没有 CSV 模板时通过对话收集数据，最后降级到行业默认值。但使用默认值时数据信心等级为 C，报价精度会降低，建议仅作为粗略参考。

### Q3: 报价结果可以导出吗？

技能输出的报价方案以 Markdown 表格格式呈现，可以直接复制到文档中。报价版本管理功能可以保存历史报价快照到 `templates/quote_version_log.csv`。如需更正式的文档格式，可以配合其他技能（如 PPTX 或 XLSX 技能）生成正式报价文件。

### Q4: 偏差学习数据存储在哪里？

偏差学习数据的存储优先级如下：
1. 如果有 MCP `record_actual_cost_feedback` 工具，写入后端数据库
2. 如果没有，追加到 `templates/project_history.csv`
3. 同时在当前会话中记忆更新后的风险系数

### Q5: 可以添加新的行业知识库吗？

可以。在 `references/` 目录下按照现有知识库的格式创建新的行业知识文件（如 `automotive-knowledge.md`），并在 SKILL.md 的"Industry Extension"部分添加引用。建议参考 `equipment-knowledge.md` 的结构来编写新知识库。
